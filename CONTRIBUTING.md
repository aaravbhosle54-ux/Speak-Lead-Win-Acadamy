# Contribution Guidelines

Thank you for interest in contributing to SpeechMaster! Here's how you can help.

## How to Contribute

### 1. Report Bugs
- Check if the bug already exists
- Provide detailed description
- Include browser and OS info
- Share steps to reproduce

### 2. Suggest Features
- Explain the feature clearly
- Show use cases
- Discuss implementation ideas
- Get community feedback

### 3. Add Content
- **New Topics**: Submit age-appropriate topics
- **New Quotes**: Provide quotes with proper attribution
- **New Jokes**: Submit clean, family-friendly jokes
- **New Outlines**: Create outline structures for new speech types

### 4. Improve Documentation
- Fix typos
- Clarify instructions
- Add examples
- Update outdated info

### 5. Code Improvements
- Refactor for clarity
- Optimize performance
- Improve accessibility
- Add comments

## Content Guidelines

### Topics
- Age-appropriate and relevant
- Interesting to the target age group
- Non-controversial
- Encouraging and positive

### Quotes
- Proper attribution
- Non-political viewpoint
- No controversial subjects
- Inspiring or thought-provoking

### Jokes
- Clean and family-friendly
- No political references
- No named public figures
- Short enough to perform (< 45 seconds)

### Outlines
- Clear section structure
- Practical and actionable
- Appropriate for the speech type
- Well-organized key points

## Development Process

1. **Fork the Repository**
```bash
git clone https://github.com/aaravbhosle54-ux/Speak-Lead-Win-Acadamy.git
cd Speak-Lead-Win-Acadamy
```

2. **Create a Branch**
```bash
git checkout -b feature/your-feature-name
```

3. **Make Changes**
- Edit files as needed
- Test thoroughly
- Follow code style

4. **Commit Changes**
```bash
git commit -m "Description of changes"
```

5. **Push to GitHub**
```bash
git push origin feature/your-feature-name
```

6. **Create Pull Request**
- Describe what you changed
- Explain why the change is needed
- Reference any related issues

```javascript
const topicList = [];

  // Implementation

```

### CSS
```css
/* Use CSS variables for consistency */
color: var(--primary-color);

/* Group related properties */
.element {
  /* Display properties */
  display: flex;
  
  /* Box model */
  padding: 1rem;
  margin: 0;

```

### HTML
```html
<!-- Use semantic tags -->

  <div class="section-header">
    <h2>Topics</h2>
```

## Testing

Before submitting:
1. Test in multiple browsers
2. Test on mobile devices
3. Test all features
4. Check console for errors
5. Verify responsiveness

## Documentation

- Update README.md if needed
- Update FEATURES.md for new features
- Add comments to code
- Document breaking changes

## Pull Request Process

1. Update documentation
2. Add/update tests if applicable
3. Follow code style guidelines
4. Create clear PR description
5. Link related issues
6. Be open to feedback

## Issue Reporting

### Bug Report Template
```
**Describe the bug:**
A clear description of what happened

**Steps to reproduce:**
1. Go to...
2. Click on...
3. See error

**Expected behavior:**
What should have happened

**Screenshots/Logs:**
If applicable, add screenshots or error logs

**Environment:**
- Browser: [e.g. Chrome 90]
- OS: [e.g. Windows 10]
- Device: [e.g. Desktop]
```

### Feature Request Template
```
**Description:**
Clear description of the feature

**Use case:**
Why would this be useful?

**Suggested implementation:**
Any ideas on how to implement this?

**Alternatives considered:**
Are there other ways to solve this?
```
